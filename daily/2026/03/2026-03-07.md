# 2026-03-07 study log
- 《算法图解》——散列表（哈希表）。就是python里的字典。
  - 散列函数：将输入**映射**到数字。
  - 散列表应用：
    - 用于查找，模拟映射关系（创建电话簿、DNS解析）
    - 防止重复（记录已经投票过的人）
    - 缓存/记录数据，以免服务器再通过处理来生成它们。
  - 性能:平均情况：查找、插入、删除均为常量时间O(1)；最糟情况（冲突）：均为线性时间O(n)
  - 哈希冲突：两个不同的键映射到了同一个位置
    - 在这个位置存储一个链表——若这个链表很长，哈希表的速度急剧下降
      - 降低填装因子
        - 填装因子=散列表包含的元素数/位置总数(底层数组长度）
        - 填装因子>0.7则扩容：数组长度翻倍 重哈希（重新分配）
 - 运用：只要在读题或者构思的瞬间，脑海里闪过这样一个念头：“我是不是需要快速查找某个元素【有没有出现过】？” 或 “我是不是需要通过 A 快速找到对应的 B？”——用哈希表
   - 问“A 和 B 有没有重合部分？” ——立刻想到集合 (set) 去重求交。
   - 问“数组里谁和谁能凑成某对关系？” ——立刻想到用字典 (dict) 存索引，一边遍历一边找另一半
   - 问“这个字母/单词出现了几次？” ——立刻想到用字典 (dict) 来做计数器。
   - **集合(set)**：只关心一件事：“你在不在名单上？”——判断元素存在、求交集/并集

   - **字典（dict）**：绑定着一份额外的信息（Key-Value 键值对）——需要找配对、需要统计频率
     - Key（键）：通常是你正在处理的数字或字符
     - Value（值）：通常是这个数字的索引（位置），或者是这个字符出现的次数
     - 题目：给一个数组 [2, 7, 11, 15] 和一个目标值 target = 9。找出哪两个数字相加等于 9，返回它们的索引。
       ```python
       class Solution:
       def twoSum(self, nums: List[int], target: int) -> List[int]:
        records = dict() # 这就是红娘的登记册
        
        for index, num in enumerate(nums):
            # 1. 算出当前数字命中注定的另一半是谁
            needed = target - num 
            
            # 2. 去登记册里找，另一半在不在？
            if needed in records:
                # 在的话，牵手成功！返回另一半的索引，和自己的索引
                return [records[needed], index]
            else:
                # 不在的话，就把自己登记在册子里，耐心等待
                # 键(Key)是数字本身，值(Value)是它的索引
                records[num] = index 
                
        return []
       ```
