###[242.有效字母的异同](https://leetcode.cn/problems/valid-anagram/description/):分步式缓存
- 准备一个“字母计分板”，让第一个单词给出现的字母加分，第二个单词给对应字母减分，最后只要所有字母的分数完美抵消、全部归零，它们就是异位词
- Python 代码使用 collections.defaultdict() 类，可指定所有 key 对应的默认 value
  - 一个自带“自动补全”功能的字典。当你访问一个根本不存在的键（Key）时，它绝对不会弹红色的 KeyError 报错，而是会自动帮你把这个键建好，并塞进去一个默认值。
  ```python
  class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        if len(s) != len(t):# 1. 长度不同，直接判定为不是异位词
            return False
        dic = defaultdict(int)# 2. 准备计分板（哈希表/字典）
        for c in s:# 3. 遍历单词 s，给出现的字母加分
            dic[c] += 1
        for c in t:# 4. 遍历单词 t，给出现的字母减分
            dic[c] -= 1
        for val in dic.values():# 再遍历一遍去检查是否有非 0 值
            if val != 0:
                return False
        return True
  ```
  ### 哈希数据结构：unordered_set（C++）
  - 底层是哈希表，特点是极速 $O(1)$，但是无序
    - 只存钥匙（Key），不存值（Value）：纯粹只记录元素是否存在
    - 天生去重
    - 毫无顺序
  - 底层运作？
    - 计算 Hash 值：当你执行插入操作（比如 insert(42)）时，底层的 Hash 函数会把数字 42 算出一个巨大的哈希值。
    - 取模找桶（Bucket）：用这个哈希值对当前底层的数组长度（桶的数量）取模，算出一个具体的“房间号”（比如 5 号房）
    - 入住或排队
      - 如果 5 号房是空的，42 直接入住。
      - 如果 5 号房已经有别人了（发生哈希冲突），C++ 通常会采用**“链地址法（拉链法）”**，在 5 号房门口拉起一条链表，让 42 排在后面。
  - 用法：
    - 增（insert）：往集合里添加一个元素。时间复杂度 $O(1)$。
    - 删（erase）：从集合里踢出一个元素。时间复杂度 $O(1)$。
    - 查（find 或 count）：用来回答“这个数字在不在名单里？” 时间复杂度$O(1)$。

 - [202.快乐数](https://leetcode.cn/problems/happy-number/)
   - 题目中说了会**无限循环**，那么也就是说求和的过程中，sum会重复出现
     - 要快速判断一个元素是否出现集合里
       - 使用哈希法，来判断这个sum是否重复出现，如果重复了就是return false， 否则一直找到sum为1为止。
         ```python
         class Solution:
         def isHappy(self, n: int) -> bool:
         # 1. 我们先写一个专属小助手，专门负责计算各位数字的平方和
             def get_sum(num):
                  total_sum = 0
                  while num > 0:
                      digit = num % 10       # 剥下最后一位数字
                      total_sum += digit ** 2 # 把它的平方加进总和里
                      num = num // 10        # 砍掉最后一位，数字变短
                  return total_sum
            
                 # 2. 准备我们的“历史记录本”（集合 set）
                  record = set()
        
                 # 3. 开始命运的循环判定
                  while True:
                      n = get_sum(n)  # 召唤小助手，算出下一步的数字
            
                 # 命运A：变成 1 啦，成功通关！
                  if n == 1:
                       return True
                
                 # 命运B：这个数字在记录本里出现过！说明掉进死循环了！
                 if n in record:
                      return False
                else:
                # 如果既不是 1，也没出现过，那就把它记在账本上，继续下一轮
                     record.add(n)
         ```
       - 可使用快慢指针：
    ```python
    class Solution:
        def isHappy(self, n: int) -> bool:
            # 1. 我们依然需要这个专属小助手，负责算平方和
            def get_sum(num):
                total_sum = 0
                while num > 0:
                    digit = num % 10
                    total_sum += digit ** 2
                    num = num // 10
                return total_sum
                
            # 2. 龟兔站在起跑线准备！
            slow = n
            fast = get_sum(n)
            
            # 3. 比赛开始！
            # 只要兔子没跑到终点（1），并且还没追上乌龟，比赛就继续
            while fast != 1 and slow != fast:
                slow = get_sum(slow)            # 乌龟走一步
                fast = get_sum(get_sum(fast))   # 兔子走两步（嵌套调用两次）
                
            # 4. 比赛结束，出成绩了！
            return fast == 1
    ```
    - [梦开始的地方1.两数之和](https://leetcode.cn/problems/two-sum/description/)——03.07
      ```python
      class Solution:
      def twoSum(self, nums: List[int], target: int) -> List[int]:
        records = dict()  # 准备红娘的登记册
        
        for index, num in enumerate(nums):
            needed = target - num  # 算出需要的另一半
            
            # 如果另一半已经在册子里了
            if needed in records:
                # 直接返回俩人的索引，牵手成功！
                return [records[needed], index]
            
            # 如果不在，就把自己登记在册子里（Key是数字，Value是索引）
            records[num] = index
            
        return []
      ```
