# 日期转换题通用套路

## 一、核心思想

这类题的关键通常不是“两个格式之间直接硬转”，而是先找一个**统一的中间量**，再进行转换。

最常见的中间量是：

- 从起点开始经过了多少天
- 第几个单位
- 第几个位置
- 当前是整体序列中的第几个编号

也就是：

```text
原格式 → 统一编号（中间量）→ 目标格式
````

---

## 二、为什么要这样做

因为两种不同的表示方式，本质上描述的是**同一个对象**。

例如日期转换题中：

* 格式 A 和格式 B 都在描述“同一天”
* 那么最稳妥的方式就是先把它们都对应到“这是第几天”

这样可以避免直接在两套规则之间来回硬推，减少混乱和出错。

---

## 三、通用步骤

### 1. 找共同起点

先看题目有没有给出两种表示方式共同对应的起点。

例如：

* A 格式的某个日期
* B 格式的某个日期

它们表示同一天，这个就是“第 0 个单位”的参考点。

---

### 2. 把输入格式转成统一编号

也就是求：

```text
从起点开始走了多少个单位
```

常见单位有：

* 天
* 分钟
* 秒
* 步数
* 位置编号

---

### 3. 再把统一编号拆成目标格式

这一部分通常依赖：

* 整除 `//`
* 取模 `%`

因为很多题目本质上都有“周期循环”。

---

### 4. 注意编号从 0 还是从 1 开始

这是最容易错的地方。

例如：

* 列表下标通常从 `0` 开始
* 题目中的编号有时从 `1` 开始

所以有时候需要 `+1`，有时候不需要。

---

## 四、通用模板

### 第一步：原格式转统一编号

```text
index = 把原格式转换成从起点开始的编号
```

### 第二步：统一编号转目标格式

```text
用整除和取模，把 index 拆成目标格式的各部分
```

---

# 整除和取模在周期题里的作用

## 一、核心结论

### 整除 `//`

表示：

> 过了多少个完整周期

### 取模 `%`

表示：

> 在当前这个周期里走到了哪个位置

可以记成一句话：

> **整除看轮数，取模看位置。**

或者：

> **`//` 看过了几圈，`%` 看这一圈走到哪。**

---

## 二、最简单的理解方式

假设一周有 7 天，现在是第 20 天。

那么：

```python
20 // 7 = 2
20 % 7 = 6
```

含义是：

* 过了 `2` 个完整星期
* 当前在这一星期里的第 `6` 个位置（如果从 0 开始数）

---

## 三、在日期题里的常见作用

### 1. 整除 `//` 的作用

整除通常用来求：

* 第几年
* 第几小时
* 第几轮
* 过了多少个完整周期

例如：

```python
year = total_days // 260
```

表示：

* 每 260 天是一个周期
* 一共过了多少个完整周期

---

### 2. 取模 `%` 的作用

取模通常用来求：

* 当前周期中的位置
* 当前是星期几
* 当前轮的第几个编号
* 当前对应哪个名字 / 标签

例如：

```python
pos = total_days % 20
```

表示：

* 在 20 个名字的循环中
* 当前落在第 `pos` 个位置

---

## 四、常见搭配规律

### 情况 1：目标编号从 0 开始

如果目标本来就是从 `0` 开始编号，例如数组下标：

```python
name = names[total_days % 20]
```

这里不需要 `+1`。

---

### 情况 2：目标编号从 1 开始

如果题目里的编号是从 `1` 开始，例如 `1~13`：

```python
number = total_days % 13 + 1
```

这里必须 `+1`。

---

## 五、一个通用判断方法

做题时可以问自己两句：

### 1. 我想知道“过了多少整轮”吗？

如果是，用：

```python
//
```

### 2. 我想知道“当前轮走到哪里了”吗？

如果是，用：

```python
%
```

---

# 这道 Maya Calendar 题的套法

## 一、统一中间量

这题的统一中间量就是：

```text
total_days = 从 0. pop 0 开始经过了多少天
```

---

## 二、Haab → total_days

公式：

```text
total_days = year * 365 + month_index * 20 + day
```

解释：

* `year * 365`：前面完整年份的天数
* `month_index * 20`：这一年中前面完整月份的天数
* `day`：当前月份已经过去的天数

---

## 三、total_days → Tzolkin

### 1. Tzolkin 年份

```python
tz_year = total_days // 260
```

因为 Tzolkin 一年 260 天。

---

### 2. Tzolkin 数字

```python
tz_number = total_days % 13 + 1
```

因为数字按 `1~13` 循环。

---

### 3. Tzolkin 名字

```python
tz_name = tzolkin_names[total_days % 20]
```

因为名字按 20 个循环，且列表下标从 0 开始。

---

## 四、这题最核心的一句话

这题不要直接在 Haab 和 Tzolkin 之间硬转，而是：

```text
Haab 日期 → total_days → Tzolkin 日期
```

---

# 总结

## 日期转换题通用思路

先找到两种表示方式共同对应的起点，再把原格式统一转换成**从起点开始经过的总单位数**，最后根据目标格式的周期规则，用整除和取模还原出目标答案。

## 整除与取模的作用

* 整除 `//`：求完整周期数
* 取模 `%`：求当前周期中的位置

一句话：

> **整除看轮数，取模看位置。**
